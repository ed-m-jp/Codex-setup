# Plan Template

Use this exact structure when outputting a plan. Do not add extra sections or preamble.

```markdown
# Plan

<1-3 sentences: what we're doing, why, and the high-level approach.>

## Scope

- In:
- Out:

## Action items

[ ] <Step 1>
[ ] <Step 2>
[ ] <Step 3>
[ ] <Step 4>
[ ] <Step 5>
[ ] <Step 6>

## Open questions

- <Question 1>
- <Question 2>
- <Question 3>
```

## Section notes

- **Summary paragraph**: Keep to 1-3 sentences. State what, why, and the broad approach.
- **Scope**: Short bullet points. "In" lists what this plan covers. "Out" lists what it explicitly does not touch.
- **Action items**: 6-10 items by default. Verb-first, atomic, ordered (discovery -> changes -> tests -> rollout). See `checklist-guidance.md` for patterns.
- **Open questions**: Max 3. Only include if there are genuine unknowns that could change the approach. Omit the section entirely if there are none.
